# UnderWatch 前端口播答辩稿（精简可口播版）

## 开场
- 大家好，我来汇报 UnderWatch 智能仓储管理系统的前端设计与实现。
- 我们聚焦三点：架构与技术栈、关键交互与品牌落地、性能与可维护性。

## 技术栈与架构
- 技术栈：Vue 3 + Vite + Element Plus，TypeScript 友好，组件生态成熟。
- 架构分层：
  - 布局层：Navbar、Sidebar、AppMain 负责整体框架与路由承载。
  - 页面层：Login、Register 等业务页，承载交互与校验。
  - 通用组件层：图标、滚动容器、菜单项等复用组件。
- 状态管理：基于 Pinia 模块化拆分（用户、权限、设置、标签视图）。
- 路由权限：Sidebar 的菜单树来源于权限路由，激活态随路由变化自动高亮。

## 关键改造与品牌落地
- 品牌更新：系统名从 RuoYi 统一升级为 UnderWatch，登录与注册页品牌呈现一致。
- 守护者猫（System Guardian）：
  - 登录页引入“夜班仓库守护者”形象，SVG 眼睛跟随焦点与鼠标，交互克制。
  - 视觉主题采用工业化渐变与玻璃拟态，兼顾企业审美与可读性。
- Logo 区域统一：
  - login.vue / register.vue 均使用猫标识图片，改造为“充满容器”的呈现。
  - 关键样式：position: absolute + object-fit: cover，使图片填满父容器，提升视觉张力。
- 侧边栏满高：
  - Sidebar 容器高度调整为 100vh，内部 el-scrollbar 与 el-menu 100% 占满。
  - 有 logo 时自动减去 50px，高度计算一致，滚动体验稳定。

## 交互体验与无障碍
- 表单聚焦态：输入框获得焦点时高亮边框与图标色，给用户明确路径。
- 键盘操作：登录与注册支持 Enter 提交，验证码区块独立交互。
- 对比度与字号：主要文案采用高对比中字号，保证在弱光环境的可读性。
- 动效约束：浮动与跟随动效帧率适中，避免过度动画影响操作效率。

## 性能与工程实践
- Vite 冷启动快，按需编译，开发体验流畅。
- 资源优化：SVG 图标复用，长列表菜单使用原生滚动与 el-scrollbar，减少重绘。
- 样式管理：主题色、阴影、圆角等统一抽象为 CSS 变量，便于全局换肤与统一风格。
- 代码质量：样式作用域、模块化 store、组件解耦，降低维护成本。

## 常见问答
- 为什么选 Vue 3 + Element Plus？
  - 生态成熟、社区稳定、组件可达生产级；与现有团队技术栈匹配。
- 守护者猫是否影响性能？
  - 交互逻辑为轻量原生计算与 SVG 动效，帧率与重绘可控，未引入重型库。
- 如何扩展到更多主题？
  - 色彩、圆角、阴影均抽象为变量，换肤只需调整主题变量文件。
- 可维护性如何保证？
  - 组件职责单一、状态模块化、样式作用域隔离，支持渐进式迭代。

- UnderWatch 的前端实现在品牌、交互与工程质量上保持平衡。
- 我们用一致的视觉系统与稳健的架构，支撑仓储业务的高效与可靠。
